# AI Security and Prompt Injection Learning Path

A practical learning path for developers, AppSec teams, and AI builders who want to understand prompt injection, AI-generated code risks, and secure AI application development.

## Who Is This For?

This learning path is designed for:

- Developers building AI-powered applications
- AppSec teams reviewing LLM applications
- AI product teams
- Security champions
- Engineering managers adopting AI coding tools

## What You Will Learn

By following this path, you can learn how to:

- Understand prompt injection risks
- Review prompts with a security mindset
- Identify unsafe AI-generated code patterns
- Recognize insecure LLM application design patterns
- Connect AI security issues to secure coding practices

## Suggested Order

1. Start by learning what prompt injection is.
2. Test example prompts with the Prompt Injection Tester.
3. Review common unsafe prompt patterns.
4. Study related application security vulnerabilities.
5. Review AI-generated code for insecure assumptions.
6. Build a checklist for secure AI-assisted development.
